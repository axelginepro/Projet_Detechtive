1. Contexte & Objectifs

Scénario : Pilotage d'un projet d'infrastructure global : conception de l'architecture réseau sécurisée et intégration complète de la stack technique (Web, Data, Fichiers) dans une approche Secure by Design.

  • Confidentialité : Chiffrement intégral des flux (HTTPS/SSL), segmentation stricte par VLANs et sécurisation du stockage par cloisonnement[cite: 57, 405].
  • Disponibilité : Architecture robuste — grâce au cloisonnement VLAN, la compromission d'un élément ne propage pas l'attaque au reste du réseau[cite: 57, 188].
  • Traçabilité : Centralisation des logs (Wazuh SIEM) depuis un VLAN Management dédié et authentification unique (Active Directory)[cite: 132, 398].

2. Architecture Infrastructure

Segmentation réseau — 5 zones de sécurité

▶ Architecture : Router-on-a-Stick (802.1Q)

Le switch IOU opère en L2 pur (commutation uniquement, pas de routage inter-VLAN)[cite: 59]. pfSense centralise l'ensemble du routage, du filtrage et du NAT via 5 sous-interfaces 802.1Q sur son interface LAN physique[cite: 75, 77]. Un trunk 802.1Q unique relie le switch à pfSense — pfSense porte toutes les passerelles VLAN[cite: 34].

Pourquoi cette segmentation ?

Serveur Web en DMZ

Le Web Server (.10) est le seul élément exposé. En l'isolant dans VLAN 10, une compromission n'expose ni l'AD, ni la BDD[cite: 125, 187].

AD isolé (VLAN 40)

L'Active Directory (.34) est le composant critique. Un attaquant en DMZ ne peut pas atteindre l'AD — flux VLAN 10 → 40 bloqué[cite: 171, 293].

Wazuh isolé (VLAN 50)

Le SIEM Wazuh (.42) est en zone dédiée. Même si la DMZ est compromise, l'attaquant ne peut pas désactiver la supervision[cite: 186, 398].

3. Gestion des Identités (Active Directory)

Moindre Privilège (Least Privilege)

Le principe fondamental de sécurité appliqué au projet[cite: 109].

  • Structure AD : Organisation par OU pour séparer les Agents des Administrateurs.
  • Droits Restreints : Les utilisateurs standard n'ont aucun droit d'administration sur les serveurs ou postes.
  • Comptes de Service : L'application Web utilise un compte AD dédié avec des droits minimaux.
  • PKI Interne (AD CS) : Le serveur AD gère l'Autorité de Certification pour chiffrer les flux internes (Web/DB)[cite: 405].

Serveur de Fichiers Sécurisé

Isolation stricte des données sensibles.

  • Dossiers Dédiés : Chaque agent dispose d'un dossier partagé personnel sur le File Server (.20, VLAN 20).
  • Permissions NTFS : ACLs configurées pour l'isolation complète entre agents.
  • Mappage Dynamique : Au login, seul le lecteur réseau correspondant est monté via GPO.

4. Développement & Backend (Détail)

Apache (XAMPP/Windows) PHP 8.x MariaDB Windows Server 2022 SMB / 445 PDO + SSL

Serveur Web — VLAN 10

IP fixe : 192.168.10.10. Seul point exposé à Internet via NAT[cite: 122, 149].

  • Rôle Pivot : Orchestre les demandes vers la BDD (3306) et le File Server (445) en VLAN 20[cite: 153, 162].
  • Sessions : Gestion stricte via session_start() et redirection si non authentifié.
  • Sécurité : Validation des entrées et whitelist des fichiers uploadés.

Base de Données — VLAN 20

IP fixe : 192.168.10.21. Inaccessible depuis Internet[cite: 195, 207].

  • Connexion SSL : Le client PHP utilise PDO::MYSQL_ATTR_SSL_CA pour valider l'identité MariaDB.
  • Isolation : VM dédiée en VLAN 20 — aucun accès direct depuis DMZ hors port 3306[cite: 145].
  • Failover : try/catch PHP pour basculer en mode dégradé si SSL échoue.

File Server — VLAN 20

IP fixe : 192.168.10.20. Stockage des preuves en zone protégée[cite: 159, 239].

  • Protocole : Partage Windows (SMB port 445) monté dynamiquement depuis VLAN 10[cite: 154].
  • Bridge PHP : Commande net use pour monter le lecteur réseau à la demande.
  • Permissions : Compte de service AD dédié (Least Privilege) — écriture sur dossier restreint.

5. Sécurité Applicative (AppSec)

Protection du Code

  • Anti-SQLi : Usage exclusif de PDO et des requêtes préparées.
  • Anti-RCE : Renommage forcé des fichiers uploadés (preg_replace) pour bloquer les scripts malveillants.
  • Feedback SSL : Indicateur visuel informant l'agent de l'état du chiffrement via Ssl_cipher.

Sécurité Infrastructure

  • Switch IOU (L2) : Commutation pure — Trunk 802.1Q unique vers pfSense. Ports inutilisés en VLAN 999 + shutdown[cite: 53, 59].
  • Pare-feu pfSense : Routage inter-VLAN centralisé, filtrage stateful, NAT Web et VPN WireGuard UDP 51820[cite: 113, 122].
  • SIEM Wazuh : Alertes temps réel depuis VLAN 50 dédié — inaccessible depuis la DMZ[cite: 132, 186].
  • Traçabilité : Accès Internet autorisé mais intégralement loggué vers Wazuh pour détecter l'exfiltration[cite: 388, 389].
  • Hardening : Désactivation des services inutiles et isolation des ports (VLAN 999 shutdown)[cite: 53, 403].

6. VPN WireGuard (Accès Admin Distant)

Paramètres

  • Protocole : UDP, Port 51820 sur WAN pfSense[cite: 321, 322].
  • Plage VPN : 10.10.10.0/24 (réseau virtuel WireGuard)[cite: 323].
  • Accès autorisé : VLAN 40 (AD) et VLAN 50 (Management)[cite: 333, 339].
  • Accès interdit : VLAN 10 (DMZ) et VLAN 30 (Clients) — principe du moindre privilège[cite: 352, 359].

Justification

Le VPN est réservé aux administrateurs IT en télétravail. Il permet de gérer pfSense, l'AD et Wazuh sans exposer RDP/SSH directement sur Internet[cite: 117, 367]. Les agents accèdent au réseau physiquement depuis le VLAN 30.
